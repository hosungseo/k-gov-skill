# public-data-portal

## 목적

공공데이터포털 API를 에이전트가 쓰기 쉽게 준비하는 스킬입니다.

## 사용자 신청/로그인

필요/선택.

공식 API 호출에는 사용자가 공공데이터포털에서 직접 활용신청한 serviceKey가 필요합니다. 키가 없으면 API 후보 탐색, 신청 방법 안내, 샘플 데이터 기반 설명까지만 수행합니다.

## 필요한 환경변수

```bash
DATA_GO_KR_SERVICE_KEY="..."
```

## 신청처

- https://www.data.go.kr
- 검색어 예: `행정안전부 인구`, `국토교통부 실거래`, `학교알리미`, `공동주택`

## 사용 예시 프롬프트

```text
공공데이터포털에서 행안부 주민등록 인구 관련 API를 찾아줘.
내가 serviceKey를 발급받아야 하는지, 필요한 파라미터가 뭔지, 호출 예시와 결과를 업무보고용 표로 정리해줘.
```

## 예상 산출물

- API 후보명
- 제공기관
- 활용신청 필요 여부
- 필요한 환경변수
- 주요 파라미터
- 호출 예시
- 키 없을 때 대체 경로
- 출처/기준일

## 키 없을 때 fallback

- 신청 방법 안내
- 공개 문서/샘플 데이터 기반 mock output
- 기존 공개 코퍼스 또는 GitHub 데이터가 있으면 그쪽으로 우회

## 주의

- 개인정보가 포함될 수 있는 API는 집계값 중심으로 사용합니다.
- 응답에는 기준일과 제공기관을 반드시 남깁니다.
- 대량 호출은 rate limit과 이용약관을 확인합니다.
