# org-quota-review

부처의 기구·정원 요구자료를 검토 가능한 업무 패킷으로 바꾸는 스킬입니다.

## 사용자 신청/로그인

선택.

기본 검토는 로컬 문서와 공개 법령/관보/보도자료 코퍼스로 수행합니다. 국회·공공 API 실호출에는 별도 key가 필요할 수 있습니다.

## 필요한 환경변수

```bash
LAW_GO_KR_OC="..."          # 선택
ASSEMBLY_API_KEY="..."     # 선택
DATA_GO_KR_SERVICE_KEY="..." # 선택
```

## 사용 예시 프롬프트

```text
OO부 정원 증원 요구자료를 검토 패킷으로 만들어줘.
요구사항, 법령 근거, 유사 전례, 기능 중복 가능성, 국회 질의 리스크, 보완요구 후보를 나눠줘.
```

## 예상 산출물

- 요청 요약
- 요구 정원/기구 변화
- 법령·직제 근거
- 관보/전례 후보
- 기능 중복·업무량 쟁점
- 보완 요구사항
- 반대검토
- 검토의견 초안

## 키 없을 때 fallback

- 공개 자료/로컬 파일 기반 검토
- 필요한 API key와 신청 방법 안내
- 확인하지 못한 항목을 `사람 검토 필요`로 표시

## 주의

- 최종 판단/승인/대외 회신은 사람 검토가 필요합니다.
- 정원·조직 검토는 숫자보다 근거, 전례, 기능 연결이 중요합니다.
